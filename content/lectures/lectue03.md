# Lecture 03 — Техники тест-дизайна

**Тема:** Методы, с помощью которых создаются эффективные тесты без "слепого" перебора всех возможных вариантов.
**Цель урока:** Изучить основные техники тест-дизайна: классы эквивалентности, анализ граничных значений и таблицы принятия решений.
**Время:** ~20 мин.
**Ключевые понятия:** Тест-дизайн, тестовые сценарии, классы эквивалентности, граничные значения, таблица принятия решений.

### 1) Введение
Невозможно протестировать программу со всеми возможными значениями входных данных — это займет бесконечное количество времени. Техники тест-дизайна позволяют выбрать минимальное, но наиболее эффективное множество тестов, которое с высокой вероятностью найдет большинство ошибок.

### 2) Основная часть
**Классы эквивалентности (Equivalence Partitioning):**
Мы разбиваем все возможные входные данные на группы (классы). Если тест работает правильно для одного значения из класса, то он будет работать правильно и для всех других значений этого класса. И наоборот, если находит ошибку для одного значения, то найдет для всех.
*   *Пример:* Поле "Возраст" принимает значения от 18 до 60 лет. Классы эквивалентности: 1) меньше 18 (невалидный), 2) от 18 до 60 (валидный), 3) больше 60 (невалидный).

**Анализ граничных значений (Boundary Value Analysis):**
Больше всего ошибок скрывается на границах классов. Поэтому мы тестируем именно граничные значения.
*   *Пример:* Для того же поля "Возраст" (18-60) мы проверим значения: **17, 18, 60, 61**.

**Таблица принятия решений (Decision Table):**
Эта техника используется, когда результат зависит от комбинации нескольких условий. Она помогает учесть все возможные комбинации "Да/Нет".
*   *Пример:* Условия: 1) Клиент зарегистрирован? 2) Товар в наличии? Результат: "Можно купить" = Да, только если выполнены оба условия.

### 3) Пример/практика
Допустим, есть поле для ввода количества символов в пароле: от 8 до 16 включительно.
1.  Определите классы эквивалентности.
2.  Определите значения для анализа граничных значений.
3.  Какие значения вы бы ввели для теста?

**Задание:**
Предложите набор тестовых значений (4-5 чисел), используя две описанные техники.
